# 20일차: 크론탭(Crontab)을 활용한 모니터링 자동화 / 스케줄링 (2026.03.23)

## 오늘의 목표
- 리눅스 스케줄러인 Crontab의 사용법을 익히고 자동화 환경을 구축한다.
- 19일차에 작성한 `sys_check.sh`를 1분 단위로 자동 실행하여 실시간 로그 시스템을 완성한다.

---

## 실습 기록 상세

### [실습 1] 크론탭(Crontab) 예약 작업 설정 및 권한 부여
`crontab -e` 명령어를 실행하여 편집기에 진입한 뒤, 매 분마다 스크립트가 실행되도록 `* * * * *` 설정을 추가했습니다. `chmod +x` 명령어로 스크립트 파일에 실행 권한을 부여하여 시스템 데몬이 정해진 시간에 자동으로 스크립트를 호출할 수 있도록 세팅했습니다.

### [실습 2] 스크립트 실행 및 실시간 자동 로그 출력 확인
`tail -f` 명령어를 사용하여 `sys_report.txt` 파일에 1분 간격으로 데이터가 쌓이는지 확인했습니다. 관리자가 직접 실행하지 않아도 서버가 스스로 리소스 상태를 체크하고 기록하는 자동화 모니터링 체계의 정상 동작을 검증했습니다.

## 핵심 명령어 요약
- **`crontab -e`**: 현재 사용자의 예약 작업(Cron Job) 리스트 편집
- **`* * * * * [스크립트 경로]`**: 매 분마다 작업을 실행하는 스케줄링 설정
- **`chmod +x sys_check.sh`**: 스크립트 파일에 실행 권한 부여
- **`tail -f [파일명]`**: 파일 끝에 추가되는 내용을 실시간으로 모니터링

---

## 배운 점 & 트러블슈팅
- **환경 변수의 이해**: 크론탭은 최소한의 환경에서 실행되므로 반드시 절대 경로를 사용해야 에러가 발생하지 않는다는 실무적 지식을 습득함.
- **입출력 흐름의 이해**: tail -f /home/growth/sys_check.sh은 내가 작성한 내용을 보는것이고 tail -f /home/growth/sys_report.txt를 해야하는것을 깨달음.
